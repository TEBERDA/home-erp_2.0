**Home ERP** — это комплексная экосистема для управления вашим домом. Это не просто список покупок, а полноценная ERP-система для домохозяйства, которая автоматизирует рутину и помогает поддерживать порядок в запасах, технике и делах.

---

## 🌟 Ключевые модули

### 👥 Мульти-арендность и Домохозяйства
*   **Изоляция данных**: Каждое домохозяйство — это отдельный защищенный контур.
*   **Система инвайтов**: Создайте дом и пригласите близких по короткому коду или ссылке.
*   **Профили пользователей**: Индивидуальный учет действий каждого члена семьи.

### 📦 Умный склад (Inventory)
*   **FIFO Списание**: Система автоматически списывает товары с ближайшим сроком годности.
*   **Многоуровневый учет**: Поддержка различных единиц измерения (кг, гр, литры, упаковки) с автоматической конвертацией.
*   **Локации**: Храните сахар в "Пантри", а молоко в "Холодильнике" — система подскажет, где что лежит.
*   **Инвентаризация**: Быстрая сверка физических остатков с системой.

### 🍳 Кулинарный помощник (Recipes)
*   **Интеграция с запасами**: Рецепты "видят" остатки в реальном времени.
*   **Авто-расчет**: Система подсветит, каких ингредиентов не хватает для приготовления блюда.
*   **В один клик**: Добавляйте все недостающие ингредиенты в список покупок одной кнопкой.

### 🛠️ Техника и Оборудование (Equipment)
*   **Цифровой паспорт**: Храните серийные номера, даты покупки и гарантийные сроки.
*   **Обслуживание**: Напоминания о замене фильтров, чистке или плановом ТО.
*   **Батарейки**: Отдельный трекинг для всех элементов питания в доме (пульты, датчики, весы).

### 🧹 Домашние дела (Chores)
*   **Периодические задачи**: Настройте график уборки или полива цветов.
*   **Списание расходников**: Выполнение дела может автоматически списывать бытовую химию (например, 50мл средства для пола).

### 📜 Лента активности (Activity Feed)
*   **Прозрачность**: Видьте, кто и когда купил продукты, выполнил дело или приготовил ужин.
*   **История**: Полный аудит всех изменений в доме.

## 🚀 Технологический стек

*   **Core**: Python 3.11, FastAPI
*   **ORM**: SQLAlchemy 2.0 + Alembic (Migrations)
*   **UI**: Jinja2 Templates + **HTMX** (для SPA-подобного взаимодействия без JS-фреймворков)
*   **Design**: **Tabler UI Kit** (на базе Bootstrap 5)
*   **Security**: JWT (Jose) + HTTP-only cookies + Passlib (bcrypt)
*   **Database**: PostgreSQL

## 🛠️ Установка и запуск

### 🐳 С использованием Docker (Рекомендуется)

1.  Клонируйте репозиторий.
2.  Создайте файл `.env` на базе `.env.example`.
3.  Запустите контейнеры:
    ```bash
    docker-compose up -d --build
    ```
4.  Приложение доступно по адресу `http://localhost:8000`.

### 🐍 Локальный запуск для разработки

1.  **Подготовка БД**:
    ```bash
    docker-compose -f docker-compose.dev.yml up -d
    ```
2.  **Окружение**:
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate
    pip install -r requirements.txt
    pip install -e .
    ```
3.  **Миграции**:
    ```bash
    alembic upgrade head
    ```
4.  **Запуск**:
    ```bash
    uvicorn app.main:app --reload
    ```

## ⚙️ Конфигурация (.env)

| Переменная | Описание | Значение по умолчанию |
| :--- | :--- | :--- |
| `POSTGRES_USER` | Пользователь БД | `postgres` |
| `POSTGRES_PASSWORD` | Пароль БД | `postgres` |
| `POSTGRES_DB` | Имя базы данных | `home_erp` |
| `SECRET_KEY` | Ключ для JWT (Обязательно смените!) | `super-secret-key` |
| `EXPIRING_SOON_DAYS` | За сколько дней считать товар "скоро истекающим" | `7` |

## 📂 Структура проекта

```text
├── alembic/              # Миграции базы данных
├── app/
│   ├── api/              # Маршруты (API v1 и HTML Pages)
│   ├── core/             # Конфигурация, Безопасность, Auth
│   ├── crud/             # CRUD операции (Бизнес-логика)
│   ├── db/               # Сессии SQLAlchemy и Base Model
│   ├── models/           # Модели базы данных
│   ├── schemas/          # Pydantic схемы (DTO)
│   ├── static/           # CSS, JS, Images
│   └── templates/        # HTML шаблоны (Jinja2 + HTMX)
├── uploads/              # Директория для документов техники
└── pyproject.toml        # Зависимости и настройки проекта
```
